# cbindgen — Header Generation for sfs-ffi

## What cbindgen does

[cbindgen](https://github.com/mozilla/cbindgen) inspects a Rust `#[no_mangle]`
public symbol set and generates a C (or C++) header file.  It is run either as
a build-script step (`build.rs`) or as a standalone CLI tool (`cargo cbindgen`).

cbindgen is particularly useful when:
- The set of exported symbols is large or changes frequently.
- Enums / structs need to be mirrored in C with exact field layout.
- The header generation is part of a verified CI pipeline.

## Decision for sfs-ffi: hand-written header

`crates/sfs-ffi/include/sfs.h` is hand-written rather than generated by
cbindgen for the following reasons:

1. **Surface is small and stable.**  sfs-ffi exports ~13 functions and one
   opaque `typedef struct SfsHandle SfsHandle;`.  The ABI will not change
   frequently; hand maintenance is tractable.

2. **No public struct layout crossing the boundary.**  `SfsHandle` is fully
   opaque (a forward declaration); callers only hold `SfsHandle *`.  cbindgen's
   main advantage — copying struct field layout into C — is not needed here.

3. **No build-time cbindgen dependency.**  Adding cbindgen as a `build.rs`
   dependency would require every developer and CI runner to have it installed.
   Keeping the header hand-written removes that friction.

4. **Richer hand-written documentation.**  cbindgen comments are derived
   from Rust `///` doc-strings and can be somewhat terse.  The hand-written
   header carries ownership, lifetime, and thread-safety prose that is easier
   to maintain directly in C.

## When to switch to cbindgen

Switch when:
- The exported surface grows substantially (>25 functions).
- Shared enums / structs with defined C layout are added.
- The generated-vs-actual drift is caught by a CI check (e.g., run
  `cbindgen --verify` and fail if the output differs from the committed header).

## cbindgen quick reference

```toml
# Cargo.toml (workspace or crate)
[build-dependencies]
cbindgen = "0.27"
```

```rust
// build.rs
fn main() {
    let crate_dir = std::env::var("CARGO_MANIFEST_DIR").unwrap();
    cbindgen::Builder::new()
        .with_crate(crate_dir)
        .with_language(cbindgen::Language::C)
        .generate()
        .expect("Unable to generate bindings")
        .write_to_file("include/sfs.h");
}
```

```bash
# CLI usage (install once)
cargo install cbindgen
cbindgen --crate crates/sfs-ffi --output crates/sfs-ffi/include/sfs.h
```
